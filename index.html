About us!

Social media apps such as Instagram are enabling people to more easily share their content to a wider group of people. Advances in technology have led to increased accessibility; as costs decrease, the photo quality available to amateurs increases.

Amateur photographers are now able to do a lot more, for a lot less money. This is a rapidly innovating market, with devices coming on the market that enable a wider range of photography techniques to be harnessed more easily and cheaply. One aspect of photography that is still dominated by professionals is time-lapse photography. Time-lapse photography can be used in conjunction with translational and rotational camera movement to create truly stunning time-lapses. For professionals there are belt-driven sliders that can translate a camera. However, these are prohibitively expensive for an amateur photographer.

There is no camera equipment that is reasonably priced and portable for amateurs. Motion Lapse will be able to give this fuctionality to amatuers.

The Product

This design incorporates three innovative features. The Smart-Cart has bluetooth connectivity allowing the user to easily control the cart with their phone. The ingenious Flex-Rail will allow the user to make the path that the cart will travel along, allowing for new interesting camera perspectives. The final feature is the triangulation of the wheels, they are engineered to ensure strong adhesion to the rail and smoothness for perfect time-lapses everytime.

The Smart-Cart is an innovative new design ensuring easy use and bluetooth connectivity.
The Flex-Rail allows for a programmable path in which time-lapse will be taken.
Triangulation of the wheels ensure a smooth translation and strong adherence to the rail.

Tools used for this website are predominately Atom and GitHub. Atom has provided a very easy to use source code editor and has embedded Git Control. This allows for the seemless updating of the live website which is hosted on GitPages. All what is needed is to push the master branch of the repository to GitHub. GitHub automatically uses the master branch of the repository to publish the site. Additionally Git allows for verison control meaning that if something does go wrong it is easy to go back to a previous commit and carry on working, rather than starting from the beginning again. Snippets of code (e.g. the contact form) were taken from various places such as w3schools and getbootstrap, this meant that the website could be "hacked" together without having in-depth knowledge of JavaScript, HTML or CSS.

Websites

A good website must have a ensure that the user experience is clear, easy, alluring, interesting, enjoyable and informative. The purpose of the website should be abunantly clear from the outset, and it shoud be visually attractive. The content is also very important. It must be informative but not bland and over-detailed. It should only include the relevant facts required by the intended user of the website. The navigation between pages are sections should be intuitive. It shouldn't be confusing to move through the website. In the end the user should trust the website. In essence a good website should have a great user experience and user interface design. It shouldn't be cluttered (some negative space), simple, a strong theme/brand, fluid resizing the grid dependent on window size, flexible visuals and fast loading times.

The accessibility of this website is focused around a continuous page (script-like) rather than having many multiple page. This allows for a story of our product to be told rather than giving nuggets of information on different pages. This was done to make the product more alluring to a potential customer, additionally the use of a youtube video allows the customer to get a grasp of the use of the product without neccessarily reading. With regards to blind or partially sighted people this device would not be best enjoyed by them (it takes images), but some functionality should be added to ensure that is adherence to UK Website Accessibility Laws. The website can be easily magnified and screen readers can be used to turn the text of the website into speech. For photos the captions are accurate enough to ensure that a visually impaired person would be able to understand what the image showed without seeing it. This is neccessary to abide by laws.

With regards to adherence to GDPR, data from the contact form needs to be kept in accordance to GDPR guidelines. Additionally, cookies are not used on the website, but if they were then there would need to be a banner to inform the user that they are being used. If data is collected you would have inform the user how you would intend to use the data and insure that they have been given the terms and conditions of the website. The website must ensure that the data is kept and it able to be retrieved if the user wanted to know what information was held about them.
